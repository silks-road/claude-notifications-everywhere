# Claude Notifications — browser extension

Desktop notifications when Claude finishes or needs you in **claude.ai** (browser), using the same notification pipeline (titles, four sounds, click-to-open) as the rest of `claude-notifications-everywhere`.

## How it works

```
claude.ai tab                          your Mac
┌────────────────────────┐   POST     ┌───────────────────────────┐
│ content.js              │  localhost │ claude-notifications serve │
│  watches "Stop response"│ ─────────▶ │  (LaunchAgent, port 52741) │
│  button → turn complete │  + token   │  → notification pipeline   │
│ background.js forwards   │            └───────────────────────────┘
└────────────────────────┘
```

The extension only talks to `127.0.0.1` on your own machine; a shared token (generated by the installer) authenticates it. Nothing leaves your computer.

## Setup (macOS)

1. **Start the local listener** — in a terminal, or paste into a Claude chat and let Claude run it:
   ```
   claude-notifications install-browser-listener
   ```
   It prints a **token** and installs a LaunchAgent that keeps the listener running.

2. **Load the extension in Chrome:**
   - Go to `chrome://extensions`
   - Turn on **Developer mode** (top-right)
   - Click **Load unpacked** and select this `extension/` folder

3. **Paste the token:** click the extension's icon → paste the token → **Save & test**. You should get a test notification.

Done — now every claude.ai turn notifies you, and clicking the notification opens that exact chat.

## Notes

- **Receive + click only.** Browser notifications open the chat on click; there are no approval buttons here (those are macOS-app-specific).
- **Privacy:** the listener binds to loopback and requires the token; the extension's host permissions are limited to `claude.ai` and the local port.
- **Not on the Chrome Web Store yet** — this is the unpacked/developer install. A Web Store listing may come later.
